People don't like Javascript because it is a loosely typed language. In other words, it's very loose on things.

3 Ways of Declaring Variable:

  • var
  • let
  • const

Var

Var has no respect for scope

Global scope is the Window object

Curly brackets make a new scope

There is not only scope, but there are different kinds of scope

Variables inside a function are private variables

Var respects only function scope not block scope

With scope you can always look up to find a value, but never down

Let

Respects block and function scope

With "let" the only way you can access it is within the same scope

"Var" and "Let" work more similar to each-other than "Const"

Const

"Const" is the most restrictive of the 3.

Array Methods

.reverse() will switch the order, of an array, to be backwards. This will change the actual state of the array.

.pop() will remove the end of the array and will change the actual state of the array.

.join() will make an array a string. It will NOT change the actual state of the array.

.slice([first index], [second index]) will pull everything from the [first index] value to right before the [second index] value. This does not change the state of the array.

.splice([first index], [how many objects removing], '[what is being added]') will allow you to insert and delete things from an array. The [first index] says where you want to insert an object or where to start removing objects. [How many objects removing] specifies the number of objects you want to remove. [What is being added] is the string or number you want to add at the [first index]. This will change the state of an array.

.sort() will sort the array. It is a loop that sorts based on the first digit in each number.

ex.
[1, 10, 234, 6, 9]
will be the same after sort
[1, 10, 234, 6, 9]

Because the first numbers are in order. For Will change the state of the array. It will also alphabatize an array. .sort() can also take an argument. ex.

arr.sort ( ( current, next) => current - next)

You can also sort array objects based off of their properties.

arr.find( [obj] => [conditions]). If you want to find every item, you will need to loop through again.

  • "Entity"

        An entity is a blueprint to make a new thing

        It's a model

        Practices single responsiblity

    1. Always start with an Id.
    2. List properties of thing

  • "Relationship"

        Connect relationships between different entities. if entities rely on eachother, define that relationship.

        Flow is one -> many.

        If an Entity is being used with multiple entities, then you represent that with "crows' feet"

         If we know that a key is going to be a "string" then say it. ex.

            {
                name: string,
             }

        An entity's Id is known as it's "Primary-Key" represented as "PK"

        An Id that is being referenced in another entity is a "Foreign-Key" because the entity with that Id already exists. Represented as "FK".

        An entity can only have ONE "Primary-Key", but MULTIPLE "Foreign-Keys"

        Entity Id's will NEVER change, or be given to multiple Entities.
